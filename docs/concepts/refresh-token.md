# What is Refresh Token

<LastUpdated/>

AccessToken and IdToken are [JSON Web Token](https://tools.ietf.org/html/rfc7519), and their **validity** is usually short. Usually, users need to carry AccessToken when obtaining resources. When AccessToken expires, users need to obtain a new AccessToken.

**Refresh Token** is used to obtain a new AccessToken. This can shorten the expiration time of AccessToken to ensure security, while not requiring users to log in again due to frequent expiration.

When a user authenticates for the first time, Refresh Token will be returned together with AccessToken and IdToken. Your application must store Refresh Token securely. It is as **important** as a password because Refresh Token can keep users logged in all the time.
